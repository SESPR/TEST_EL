Это тестовое задание на Laravel с загрузкой данных из эндпоинтов 'incomes', 'orders', 'sales' и 'stocks'. Для каждого эндпоинта сделана отдельная artisan-команда:

- php artisan app:get-stocks
- php artisan app:get-incomes
- php artisan app:get-sales
- php artisan app:get-orders

Вместо того, чтобы полностью стянуть данные, я постарался сделать команду многоразовой, поэтому сохраняю данные через 'upsert' чтобы избежать дубликатов при повторном запуске. 
Для 'incomes' удалось добиться полного совпадения количества записей эндпоинтом, для 'stocks' почти полного. В 'orders' и 'sale' полного совпадения нет, потому, что в самом эндпоинте не оказалось надежных уникальных ключей: часть записей дублируется или отличается только полями, которые приведут к заметному усложнению ключей или же их нельзя безопасно использовать в составном ключе.
