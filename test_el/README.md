Это тестовое задание на Laravel с загрузкой данных из эндпоинтов 'incomes', 'orders', 'sales' и 'stocks'. Для каждого эндпоинта сделана отдельная artisan-команда.

Я постарался сделать загрузку многоразовой, поэтому сохраняю данные через 'upsert'. Для 'incomes' удалось добиться полного совпадения количества записей с API, для 'stocks' почти полного. В `orders` и `sales` полного совпадения нет, потому что в самом API не оказалось надежных уникальных ключей: часть записей дублируется или отличается только полями, которые приведут к заметному услохнению кличей или же их нельзя безопасно использовать в составном ключе.
